# CAD Stone Networks — Files Feature: What's Wrong & How BuilderTrend Does It

**Date:** April 7, 2026
**Purpose:** Replit needs to follow these instructions EXACTLY to match BuilderTrend's Files experience. This is the client's #1 feature — the whole reason this tool exists.

---

## THE PROBLEM

In our current CAD Stone build, **there is no "Files" in the main navigation at all.** Documents, Photos, and Videos are buried as tabs inside a specific job's detail page. A user has to: click Jobs → find the right job → click into it → then click the Documents/Photos/Videos tab.

In BuilderTrend, **Files is a top-level navigation item** — it sits in the main nav bar at the same level as Sales, Jobs, Project Management. It's always accessible from anywhere in the app. This is how the client accesses their files every day.

---

## WHAT BUILDERTREND DOES (Copy This Exactly)

### 1. Top-Level "Files" Dropdown in Main Navigation

**What it looks like:** In the top nav bar (the orange bar), there's a menu item called **"Files"**. When you click it, a dropdown appears with THREE options:

- Documents (with a document icon)
- Photos (with a photo/image icon)
- Videos (with a play button icon)

Each one is a separate, full page — not a tab buried inside a job.

**What to build:** Add a **"Files"** item to the CAD Stone left sidebar (between "Jobs" and "Sales"), with a collapsible sub-menu or dropdown showing:
- Documents
- Photos
- Videos

Clicking any of these should navigate to a DEDICATED full page for that file type, scoped to the currently selected job (or showing all jobs' files if no job is selected).

---

### 2. Documents Root View

**What it looks like in BT:**
- Page heading: **"Documents"**
- Toolbar (top right): **"+ Folder"** split button (blue, with a dropdown arrow for "Import"), toolbar icons ("...", settings gear, history/clock)
- Below toolbar: **"Sort by"** dropdown + filter icon
- **Grid/List toggle** (top right) — two icons, grid squares and list lines
- **"Folders"** section label + **"Select all"** checkbox
- Folder tile cards: large rectangular cards with folder name, folder icon (yellow), and action icons at bottom right (tag, home/share, "..." context menu)

**What to build for CAD Stone:**
- The current Documents tab inside job detail already has folder tiles and Sort by — GOOD
- **ADD** Grid/List toggle to Documents (it's currently MISSING — Photos and Videos have it, Documents doesn't)
- **ADD** Select all checkbox
- **ADD** "..." context menu on each folder tile (for rename, delete, move)
- **ADD** the "+ Folder" button should be a split button with a dropdown arrow for "Import"

---

### 3. Inside a Folder (Documents)

**What it looks like in BT:**
- The **toolbar changes completely** when you navigate inside a folder:
  - "**+ Folder**" button is GONE
  - Replaced by: **"More actions"** dropdown, **"New"** dropdown, and **"+ Upload"** button (blue, prominent)
- **Breadcrumbs**: `Documents / ** Global Documents ** / LOMBARD` — each segment is clickable to navigate back
- Folder heading shows the folder name in large text
- Empty state: document icon + "Add a document" + "Upload, annotate, and sign documents related to the job. Add a document to get started."

**What to build for CAD Stone:**
- The current folder interior view already has breadcrumbs and Upload button — GOOD
- **ADD** "More actions" dropdown (rename folder, delete folder, move folder)
- **ADD** "New" dropdown (new sub-folder)
- Make sure the breadcrumb trail is fully clickable at every level

---

### 4. Photos Root View

**What it looks like in BT:**
- Page heading: **"Photos"**
- Same toolbar pattern as Documents: "+" Folder" split button, toolbar icons
- Empty state shows a photo/landscape icon + "Add a folder" + "Organize your documents, photos, and videos. Add a folder to start uploading files."
- When photos exist inside a folder, they display as a **thumbnail grid** — actual image previews in a responsive grid layout (NOT a file table)
- Clicking a photo opens a **lightbox preview** (full-screen overlay with the image, navigation arrows for prev/next)

**What to build for CAD Stone:**
- Current Photos tab has the Grid/List toggle — GOOD
- **CRITICAL**: When photos are uploaded inside a folder, the Grid view MUST show actual image thumbnail previews in a responsive grid — not just file names in a table
- **ADD** lightbox/preview modal when clicking a photo thumbnail
- **ADD** photo count on folder tiles

---

### 5. Videos Root View

**What it looks like in BT:**
- Page heading: **"Videos"**
- Same toolbar pattern: "+ Folder" split button, Grid/List toggle, Sort by
- **Grid view (Tile)**: Shows folder tile cards with folder name and blue folder icon — same as Documents
- **List view (Table)**: Shows a table with columns: checkbox, **Name**, **Shared with**, **Modified**, **Added**, **Contents**, **Actions**
- Pagination at bottom: "1-1 of 1 items"
- When videos exist inside a folder, they display as **thumbnail cards with a play button overlay** in Grid view
- Clicking a video plays it in an inline player or modal

**What to build for CAD Stone:**
- Current Videos tab has Grid/List toggle and folder tiles — GOOD
- **CRITICAL**: When videos are uploaded inside a folder, Grid view must show video thumbnail cards with a **play button overlay icon** centered on the thumbnail
- **ADD** in List/Table view: include a **"Duration"** column (BT-specific for videos)
- **ADD** video player modal when clicking a video

---

### 6. Shared Patterns Across All Three File Sections

These patterns are IDENTICAL in Documents, Photos, and Videos in BT:

| Pattern | Description |
|---|---|
| **Grid/List toggle** | Two-icon toggle (grid squares / list lines) in top right — switches between tile cards and table view |
| **Sort by dropdown** | Appears in Grid view: "Sort by" with dropdown arrow |
| **Select all checkbox** | Appears next to "Folders" label in Grid view |
| **"+ Folder" split button** | At root level — primary action is "+ Folder", dropdown arrow reveals "Import" |
| **Toolbar changes inside folder** | At root: "+ Folder" button. Inside folder: "More actions" + "New" + "**+ Upload**" |
| **Breadcrumbs** | `Section / Parent Folder / Current Folder` — each segment clickable |
| **Folder tile cards** | Card with folder name, folder icon, action icons ("...", tag, share) |
| **Context menu ("...")** | On each folder tile — options: rename, delete, move, permissions |
| **Pagination** | "X-Y of Z items" at bottom of list view |
| **Empty state** | Relevant icon + action text + description |

## SUMMARY

The #1 thing to understand: **Files is not a sub-feature — it IS the feature.** The client chose BuilderTrend specifically for how it organizes project files into Documents, Photos, and Videos with folder structures. They want their crew to be able to pull up a job, go straight to Photos, and see every image from the kitchen install in a thumbnail grid. They want to go to Documents and find the insurance papers, the plans, the invoices — all organized in folders.

Right now, our build has the folder structure and upload capability buried in job detail tabs. That's the right data model, but the **navigation and access pattern** is wrong. Files needs to be front and center — one click from anywhere in the app, just like BuilderTrend does it.
