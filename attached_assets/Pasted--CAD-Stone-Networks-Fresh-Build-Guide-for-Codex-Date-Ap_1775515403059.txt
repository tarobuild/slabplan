# CAD Stone Networks — Fresh Build Guide for Codex

**Date:** April 6, 2026
**What this is:** Complete build instructions for a construction management tool for a ~5 person marble/stone installation company. This is an INTERNAL-ONLY tool — no clients will ever use it.
**Start from scratch.** Do not reference any previous build. Follow this document exactly.

---

## Tech Stack (use exactly these)

- **Frontend:** React 18 + TypeScript, Tailwind CSS + shadcn/ui component library, Zustand for state management, Axios for HTTP
- **Backend:** Node.js + Express.js, Drizzle ORM, PostgreSQL (Neon)
- **Auth:** Email/password with JWT (access token in memory, refresh token in httpOnly cookie)
- **Real-time:** Socket.IO
- **File Storage:** Local filesystem or S3-compatible storage
- **Deployment:** Replit

---

## CRITICAL DESIGN RULES (read before writing any UI code)

These rules are non-negotiable. The previous build failed because it ignored all of them.

1. **Use shadcn/ui components everywhere.** Buttons, inputs, selects, dialogs, tables, tabs, toasts, dropdowns, badges, cards — all from shadcn/ui. Do not hand-roll UI components.

2. **Clean, modern SaaS design.** White content backgrounds. Light gray page backgrounds (#F9FAFB). Crisp 1px borders (#E5E7EB). Subtle shadows. 8px border-radius max. Think Linear, Notion, or Google Workspace — NOT rounded blobs with beige backgrounds.

3. **ALL creation forms are modals.** Create Job = modal. Create Lead = modal. New Schedule Item = modal. New Daily Log = modal. New Folder = modal. NEVER show a create form permanently inline on the page.

4. **Information-dense layouts.** Tables should show real data columns. Pages should not have excessive whitespace or oversized cards. Content should be compact and professional.

5. **Color palette:**
   - Primary Blue: #2563EB (buttons, links, active states)
   - Success Green: #16A34A
   - Warning Yellow: #F59E0B
   - Danger Red: #DC2626
   - Neutral Gray: #6B7280
   - Borders: #E5E7EB
   - Page background: #F9FAFB
   - Card background: #FFFFFF

6. **Typography:** System font stack or Inter. 14px body text. No ALL-CAPS letter-spaced section labels.

7. **Every list/table needs:** Search bar, sort controls, pagination, empty states.

8. **Every destructive action needs:** Confirmation dialog.

9. **Toast notifications:** Top-right corner, auto-dismiss 4s. Success (green), Error (red), Info (blue), Warning (yellow). Show for: saves, deletes, uploads, errors.

## Database Schema

Create ALL of these tables. Use Drizzle ORM migrations.

```sql
-- Users
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  full_name VARCHAR(255) NOT NULL,
  role VARCHAR(50) NOT NULL DEFAULT 'crew_member',
  avatar_url VARCHAR(500),
  phone VARCHAR(20),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP
);

-- Jobs (Projects)
CREATE TABLE jobs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title VARCHAR(255) NOT NULL,
  status VARCHAR(50) NOT NULL DEFAULT 'open',
  street_address VARCHAR(255),
  city VARCHAR(100),
  state VARCHAR(2),
  zip_code VARCHAR(10),
  contract_price DECIMAL(12, 2),
  job_type VARCHAR(100),
  work_days JSON,
  projected_start DATE,
  projected_completion DATE,
  actual_start DATE,
  actual_completion DATE,
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP
);

-- Folders
CREATE TABLE folders (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title VARCHAR(255) NOT NULL,
  job_id UUID REFERENCES jobs(id) ON DELETE CASCADE,
  parent_folder_id UUID REFERENCES folders(id) ON DELETE CASCADE,
  media_type VARCHAR(50) NOT NULL,
  viewing_permissions JSON,
  uploading_permissions JSON,
  is_global BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP,
  UNIQUE(job_id, title, parent_folder_id, media_type)
);

-- Files
CREATE TABLE files (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  folder_id UUID REFERENCES folders(id) ON DELETE CASCADE,
  filename VARCHAR(255) NOT NULL,
  original_name VARCHAR(255) NOT NULL,
  file_url VARCHAR(500),
  file_size BIGINT,
  mime_type VARCHAR(100),
  uploaded_by UUID REFERENCES users(id),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP
);

-- Leads
CREATE TABLE leads (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title VARCHAR(255) NOT NULL,
  street_address VARCHAR(255),
  city VARCHAR(100),
  state VARCHAR(2),
  zip_code VARCHAR(10),
  confidence INT DEFAULT 0,
  projected_sales_date DATE,
  estimated_revenue_min DECIMAL(12, 2),
  estimated_revenue_max DECIMAL(12, 2),
  status VARCHAR(50) NOT NULL DEFAULT 'open',
  project_type VARCHAR(100),
  notes TEXT,
  lead_source VARCHAR(255),
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP
);

-- Lead Contacts
CREATE TABLE lead_contacts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  lead_id UUID REFERENCES leads(id) ON DELETE CASCADE,
  first_name VARCHAR(100),
  last_name VARCHAR(100),
  display_name VARCHAR(255) NOT NULL,
  street_address VARCHAR(255),
  city VARCHAR(100),
  state VARCHAR(2),
  zip_code VARCHAR(10),
  phone VARCHAR(20),
  cell_phone VARCHAR(20),
  email VARCHAR(255) NOT NULL,
  label VARCHAR(100),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP
);

-- Lead Salespeople
CREATE TABLE lead_salespeople (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  lead_id UUID REFERENCES leads(id) ON DELETE CASCADE,
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(lead_id, user_id)

-- Lead Tags
CREATE TABLE lead_tags (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  lead_id UUID REFERENCES leads(id) ON DELETE CASCADE,
  tag_name VARCHAR(100),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(lead_id, tag_name)
);

-- Lead Sources
CREATE TABLE lead_sources (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  lead_id UUID REFERENCES leads(id) ON DELETE CASCADE,
  source_name VARCHAR(100),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(lead_id, source_name)
);

-- Lead Attachments
CREATE TABLE lead_attachments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  lead_id UUID REFERENCES leads(id) ON DELETE CASCADE,
  file_id UUID REFERENCES files(id) ON DELETE CASCADE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(lead_id, file_id)
);

-- Schedule Items
CREATE TABLE schedule_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  job_id UUID REFERENCES jobs(id) ON DELETE CASCADE,
  title VARCHAR(255) NOT NULL,
  display_color VARCHAR(50) NOT NULL DEFAULT 'blue',
  start_date DATE NOT NULL,
  work_days INT NOT NULL,
  end_date DATE NOT NULL,
  is_hourly BOOLEAN DEFAULT FALSE,
  start_time TIME,
  end_time TIME,
  progress INT DEFAULT 0,
  reminder VARCHAR(100) DEFAULT 'none',
  notes TEXT,
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP
);

-- Schedule Item Assignees
CREATE TABLE schedule_item_assignees (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  schedule_item_id UUID REFERENCES schedule_items(id) ON DELETE CASCADE,
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(schedule_item_id, user_id)
);

-- Daily Logs
CREATE TABLE daily_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  job_id UUID REFERENCES jobs(id) ON DELETE CASCADE,
  log_date DATE NOT NULL,
  title VARCHAR(255),
  notes TEXT NOT NULL,
  weather_data JSON,
  include_weather BOOLEAN DEFAULT TRUE,
  include_weather_notes BOOLEAN DEFAULT FALSE,
  weather_notes TEXT,
  share_internal_users BOOLEAN DEFAULT TRUE,
  share_subs_vendors BOOLEAN DEFAULT FALSE,
  share_client BOOLEAN DEFAULT FALSE,
  is_private BOOLEAN DEFAULT FALSE,
  created_by UUID REFERENCES users(id),
  published_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

-- Daily Log Attachments
CREATE TABLE daily_log_attachments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  daily_log_id UUID REFERENCES daily_logs(id) ON DELETE CASCADE,
  file_id UUID REFERENCES files(id) ON DELETE CASCADE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(daily_log_id, file_id)

-- Daily Log Tags
CREATE TABLE daily_log_tags (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  daily_log_id UUID REFERENCES daily_logs(id) ON DELETE CASCADE,
  tag_name VARCHAR(100),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(daily_log_id, tag_name)
);

-- Activity Log
CREATE TABLE activity_log (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  entity_type VARCHAR(100) NOT NULL,
  entity_id UUID NOT NULL,
  action VARCHAR(100) NOT NULL,
  user_id UUID REFERENCES users(id),
  metadata JSON,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## API Endpoints (implement all of these)

### Auth
- POST /api/auth/register — { email, password, full_name } → user + tokens
- POST /api/auth/login — { email, password } → user + tokens
- POST /api/auth/logout — clear refresh token
- POST /api/auth/refresh — refresh token → new access token
- POST /api/auth/forgot-password — { email } → send reset email
- POST /api/auth/reset-password — { token, newPassword } → reset

### Jobs
- GET /api/jobs — list (paginated, filterable by status, sortable)
- POST /api/jobs — create job
- GET /api/jobs/:id — get job detail
- PUT /api/jobs/:id — update job
- DELETE /api/jobs/:id — soft delete

### Folders
- GET /api/jobs/:jobId/folders?mediaType=document&parentId= — list folders
- POST /api/jobs/:jobId/folders — create folder
- PUT /api/folders/:id — update folder (rename, permissions)

## Build Order

Build in this exact order. Each step should be fully functional before moving to the next.

1. **Project setup:** React + Express + Drizzle + PostgreSQL connection. Run all migrations. Set up shadcn/ui.
2. **Auth:** Register, Login, Logout pages. JWT middleware. Protected routes.
3. **Layout shell:** Top nav, sidebar, routing structure.
4. **Dashboard:** Stat cards, quick actions, activity feed.
5. **Jobs:** List page with table, Create modal, Detail page with Summary tab.
6. **Files — Documents:** Folder CRUD, file upload/download, grid/list toggle, breadcrumb nav, Global Documents folder, toolbar actions.
7. **Files — Photos:** Photo folders, photo upload, thumbnail grid.
8. **Files — Videos:** Video folders, video upload, Global Videos folder.
9. **Sales/CRM:** Lead list table, Create modal, Detail side panel with all tabs, Contact management, lead-to-job conversion.
10. **Schedule:** Calendar view (use react-big-calendar or similar), List view, Gantt view (use frappe-gantt or similar), Schedule item CRUD modal.
11. **Daily Logs:** Log list with cards, Create modal with weather integration (OpenWeatherMap API), publish/draft workflow, filter sidebar.
12. **Polish:** Toast notifications everywhere, confirmation dialogs on deletes, loading skeletons, error boundaries, empty states on every list, keyboard accessibility.
