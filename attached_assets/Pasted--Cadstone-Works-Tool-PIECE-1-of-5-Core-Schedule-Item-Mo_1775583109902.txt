# Cadstone Works Tool — PIECE 1 of 5: Core Schedule Item Modal Upgrade

## Context
This is an internal project management tool for Cadstone (marble/stone/countertop fabrication & installation). The backend API routes for schedule items already exist. The frontend currently has a basic "Add Schedule Item" modal with only: Title, Start Date, Work Days, Progress %, Color (limited), Assignees (checkboxes), Notes (single text field). This task upgrades that modal to full feature parity with BuilderTrend.

## What Exists Now (DO NOT break these)
- Schedule tab within Job detail page at `/jobs/:id`
- Calendar view (month only) with Calendar/List toggle
- "+ Add Item" button that opens a simple modal
- Basic schedule item CRUD

## What To Build

### 1. Upgrade the Schedule Item Modal

The modal should have two top-level tabs:
```
[ Schedule Item Details ]  [ Related Items (count) ]
```

### 2. Schedule Item Details — Main Fields

Rebuild the form with these fields in this exact layout order:

| Field | Type | Required | Details |
|-------|------|----------|---------|
| Complete | Circle checkbox | No | Only visible AFTER item is saved. Clicking marks the item done. Renders as an empty circle that fills/checks on click |
| Title | Text input | Yes | Placeholder: e.g. "Granite Countertop Template" |
| Display Color | Dropdown with color swatches | No | 16 named colors, each with a colored circle next to the name: Maroon, Merlot, Tuscan Red, Rose, Victoria, Brown, Coffee, Amber, Cucumber, Plum, Purple, Lavender, Iris, Violet, Navy, Levi (Levi = default, blue) |
| Assignees | Multi-select user picker | No | Text input with autocomplete searching existing users. Include a "send/notify" icon button next to the field |
| Start Date | Date picker | Yes | Defaults to today |
| Work Days | Number input (spinner) | Yes | Defaults to 1. Shows "day" or "days" label |
| End Date | Date picker | Yes | Auto-calculated: Start Date + Work Days (skipping non-workdays). Can also be manually overridden, which recalculates Work Days |
| Hourly toggle | Switch (on/off) | No | Default off. When toggled ON, a new required "Start Time" field appears (time picker with clock icon) between the toggle and Progress |
| Progress | Slider + number input side by side | No | Slider range 0–100. Number input shows %. Moving slider updates number and vice versa. Defaults to 0% |
| Reminder | Dropdown | No | Options: None (default), 1 Hour Before, 2 Hours Before, 4 Hours Before, 8 Hours Before, 12 Hours Before, 1 Day Before, 2 Days Before |

### 3. Sub-Tabs Below Main Fields

Below the main fields, render these sub-tabs:

**Before first save:**
```
[ Predecessors & Links ]  [ Phases & Tags ]  [ Viewing ]  [ Notes ]  [ Files ]
```

**After first save (additional tabs appear):**
```
[ Predecessors & Links ]  [ Phases & Tags ]  [ Viewing ]  [ Notes (X new) ]  [ Files ]
```

#### Sub-Tab: Predecessors & Links
- Section header: "Predecessors"
- Each predecessor row has 3 fields + delete:
  - Name: Dropdown listing all OTHER schedule items in this job (shows "There are no items to select" if only one item exists)
  - Type: Dropdown with options: "Finish-to-Start (FS)", "Start-to-Start (SS)"
  - Lag: Number input + "days" label
  - Trash icon button to delete the row
- "+ Add Predecessor" button below the rows to add new rows

#### Sub-Tab: Phases & Tags
- **Schedule Item Phase** section:
  - Phase dropdown (defaults to "Unassigned"). Populated from the phases list in Schedule Settings
  - "Add" link/button to create a new phase inline
  - "Edit" link/button to edit existing phases
- **Schedule Item Tags** section:
  - Tags text input field
  - "Add" link/button to create new tags
  - "Edit" link/button to manage existing tags

#### Sub-Tab: Viewing
- **Schedule Viewing** section:
  - "Show on Gantt" checkbox (checked by default)
  - "Visible to Estimators" checkbox (checked by default)
  - "Visible to Installers" checkbox (checked by default)
  - "Visible to Office Staff" checkbox (checked by default)

#### Sub-Tab: Notes
- Single notes stream (no sub-tabs needed — this is internal only)
- Text area to type a note
- After saving, notes display as a list: each note shows author name, timestamp, and note text
- Tab label shows count: "Notes (X new)" after saving notes

#### Sub-Tab: Files
- Section header: "Attachments"
- "Add" button to upload files
- "Create new doc" button
- Uploaded files display as a list with filename, file type icon, and remove/delete option

### 4. Related Items Tab (second top-level tab)
- Section header: "Related To-Do's"
- Message: "To-Do's available after save"
- After the item is saved: "Save and Create To-Do" button that creates a linked to-do

### 5. Modal Footer (After First Save)
- Left side: "Created by [Avatar] [Name] on [Date, Time]"
- Right side: 
  - "..." button with menu: Copy, Delete
  - Save button (primary blue) with dropdown chevron arrow. Dropdown options: "Save and close", "Save and new"

### 6. Schedule Items Display on Calendar
- After saving, schedule items must render on the calendar as **colored horizontal bars** spanning from Start Date to End Date
- Bar color matches the item's Display Color
- Multi-day items span across day cells
- Clicking a bar re-opens the Schedule Item modal for editing
- Today's date cell shows a blue circle highlight on the date number

## API Notes
- The backend already has schedule item CRUD routes. Extend the schema/API if needed to support new fields: displayColor, hourly, startTime, reminder, endDate, phase, tags, predecessors, progress (slider value), isComplete
- Notes and Files on schedule items may need their own sub-endpoints (e.g., `/api/schedule-items/:id/notes`, `/api/schedule-items/:id/files`)
