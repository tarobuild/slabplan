  2. No Content-Security-Policy Header — MEDIUM                                                                                                                                                             

                                               

  Helmet was added but CSP was not configured. Without CSP, if an XSS vulnerability is ever found, there's no browser-level defense to limit what injected scripts can do.                                  

                                                                                                                                                                                                            

  Fix: Add a CSP to your helmet config:

  helmet({

    contentSecurityPolicy: {

      directives: {                                                                                                                                                                                         

        defaultSrc: ["'self'"],

        scriptSrc: ["'self'"],                                                                                                                                                                              

        styleSrc: ["'self'", "https://fonts.googleapis.com"],

        fontSrc: ["https://fonts.gstatic.com"],              

        imgSrc: ["'self'", "data:", "https://storage.googleapis.com"],

        connectSrc: ["'self'"],                                                                                                                                                                             

      }                        

    }

  })

  3. Static HTML Doesn't Get Security Headers — MEDIUM

                                                                                                                                                                                                            

  The security headers (helmet) only apply to API routes (/api/*). The homepage HTML itself is served directly by Google Frontend without:

  - Content-Security-Policy                                                                                                                                                                                 

  - X-Content-Type-Options 

  - X-Frame-Options       

  - Referrer-Policy                                                                                                                                                                                         

                                                                                                                                                                                                            

  This means the main page can still be iframed (clickjacking) and doesn't benefit from CSP protection.

                                                                                                                                                                                                            

  Fix: Configure your hosting layer (Replit/Google Cloud Run) to add headers to static file responses, or serve the HTML through Express as well.

  4. Duplicate Strict-Transport-Security Headers — LOW                                                                                                                                                      

                                                                                                                                                                                                            

  Two HSTS headers are sent with different max-age values:

  strict-transport-security: max-age=63072000; includeSubDomains

  strict-transport-security: max-age=31536000; includeSubDomains                                                                                                                                            

  One is from the hosting layer, one from helmet. Browsers typically use the last one, but this is ambiguous.                                                                                               

                                                                                                             

  Fix: Disable HSTS in helmet since the hosting platform already sets it:                                                                                                                                   

  helmet({ hsts: false })
