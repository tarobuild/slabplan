  NEW ISSUE INTRODUCED
                      
  The in-memory rate limiter (rate-limit.ts:17) uses a global Map with a MAX_BUCKETS = 5000 cap. Pruning only
  triggers when the map is full (line 39). In a multi-instance deployment (the .replit uses autoscale), each        
  instance has its own map — an attacker can distribute requests across instances to bypass limits entirely. For
  production, this should use a shared store (Redis).
