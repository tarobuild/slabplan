     mounts /auth before auth enforcement, the backend accepts unauthenticated POST /api/auth/register,
     and the frontend exposes a public /register page that immediately authenticates the new user. If
     there is no external gate in deployment, this is an unauthorized account-creation path. artifacts/

     any authenticated user can enumerate all users, read/update/delete any job, edit global daily-log
     settings, and access org-wide dashboard/search data. artifacts/api-server/src/middleware/require-

  3. Critical Uploaded files create a same-origin stored-XSS/account-takeover path. /uploads is served

     raw fileUrl values in new tabs, so uploaded HTML can execute on the app origin and call /api/auth/
     refresh to mint an access token. artifacts/api-server/src/app.ts:42 artifacts/api-server/src/lib/

  4. High Realtime events are broadcast to every authenticated socket, not scoped by job, role, or room.
     Socket auth only verifies token validity, and every event is sent with io.emit(...). That leaks
     activity across the whole company dataset to any logged-in client that connects to Socket.IO.
     artifacts/api-server/src/index.ts:22 artifacts/api-server/src/lib/realtime.ts:20 artifacts/api-
     server/src/lib/realtime.ts:53 artifacts/api-server/src/lib/file-manager.ts:872
  5. High JWT secrets silently fall back to hard-coded defaults. If any production environment misses the
     secret vars, access, refresh, and reset tokens become forgeable. This is the kind of failure mode
     that turns a deploy/config mistake into full auth bypass. artifacts/api-server/src/lib/auth.ts:28
  6. High Password reset tokens are logged in plaintext and remain reusable until expiry. forgot-password
     writes the reset token to application logs, logger redaction does not cover it, and reset-password
     has no nonce/version invalidation after a successful reset. artifacts/api-server/src/routes/
     auth.ts:171 artifacts/api-server/src/routes/auth.ts:206 artifacts/api-server/src/lib/logger.ts:5
     artifacts/api-server/src/lib/auth.ts:24
  7. High The application performs schema DDL in normal request paths instead of through migrations.
     Daily-log and schedule handlers create or alter tables on demand, which means the runtime DB role
     needs DDL rights and first-request failures can become schema-management outages. artifacts/api-
